## How PTV Codes Work

- Every task the fleet does should serve a purpose code
- Agents tag their work with `Purpose: [P##]` in their SOUL.md
- New codes can be added anytime ("Hey, we have a new goal...")
- Codes can retire when achieved or deprioritized
- Ask "what are we doing toward P01?" to get a progress report across the fleet

## Code Lifecycle

1. **Proposed** — Human says "new goal" → Eoin/Relay captures
2. **Validated** — Both Robert AND Corinne confirm via bearings
3. **Active** — Tagged, tracked, reported on
4. **Retired** — Achieved or deprioritized, kept for history

## Monthly Check-In

Bearings `monthly_vision_check` asks: "Any north stars stale or missing?"
Both owners get the question. New codes emerge from their answers.

## Values Gate

Every purpose must pass the values test (from values.md):
- Does this help? Does it avoid harm?
- A purpose that fails this test doesn't get coded.
- P05 (Doing Good) IS the values gate — it governs all other codes.
